## Plugin Management

Claude Code includes built-in commands for discovering, installing, and managing plugins through the `/plugin` slash command in the interactive REPL.

### Installing Plugins

**In the Claude Code REPL:**

```bash
# Launch Claude Code
claude

# Install from marketplace
/plugin install <plugin-name>

# Install from specific marketplace
/plugin install <plugin-name>@<marketplace-name>

# Install from GitHub repository
/plugin install github:<owner>/<repo>

# Install from local path (for development)
/plugin install file:/path/to/plugin

# Install from Git URL
/plugin install git:https://github.com/user/plugin.git

# Install from NPM package
/plugin install npm:<package-name>
```

**Real plugin examples:**

```bash
# Install official LSP plugins
/plugin install pyright-lsp@marketplace
/plugin install typescript-lsp@marketplace
/plugin install rust-lsp@marketplace

# Install from GitHub
/plugin install github:anthropics/claude-code-plugins

# Install local plugin during development
/plugin install file:~/dev/my-custom-plugin
```

<Callout type="info">
**Installation Scope:** Plugins can be installed at different scopes using the settings.json file. User-level plugins (`~/.claude/settings.json`) are available across all projects, while project-level plugins (`.claude/settings.json`) are specific to that repository.
</Callout>

### Managing Installed Plugins

**Enable/disable plugins via settings.json:**

```json
{
  "enabledPlugins": {
    "plugin-name@marketplace": true,
    "another-plugin": false
  }
}
```

**List installed plugins:**

```bash
# In the REPL
/plugin list
```

**Update a plugin:**

```bash
/plugin update <plugin-name>
```

**Uninstall a plugin:**

```bash
/plugin uninstall <plugin-name>
```

<Callout type="warning">
**Session Reload:** After installing or updating plugins, you may need to restart your Claude Code session or run `/config` to reload plugin components.
</Callout>

### Creating Custom Components

**Custom Commands:**

Create a markdown file in `.claude/commands/`:

```bash
# .claude/commands/test.md
---
description: Run project tests
allowed-tools: Bash
---

Run the test suite for this project:

!npm test
```

Now accessible via: `/test`

**Custom Agents:**

Create a markdown file in `.claude/agents/`:

```bash
# .claude/agents/code-reviewer.md
---
name: code-reviewer
description: Reviews code for quality, security, and best practices
tools: Read, Grep, Glob, Bash
model: sonnet
---

You are an expert code reviewer...
```

Now accessible via: `/agents` interface

**Custom Skills:**

Create a directory in `.claude/skills/skill-name/` with `SKILL.md`:

```bash
# .claude/skills/my-skill/SKILL.md
---
name: my-skill
description: When to auto-invoke this skill
allowed-tools: Read, Edit
---

# Skill Instructions
What Claude should do when this skill is invoked...
```

Claude auto-invokes when request matches description.

### Plugin Configuration

**In settings.json:**

```json
{
  "enabledPlugins": {
    "pyright-lsp@marketplace": true,
    "my-custom-plugin": true
  },
  "extraKnownMarketplaces": {
    "team-marketplace": {
      "source": {
        "source": "github",
        "repo": "company/plugins"
      }
    }
  }
}
```

**Plugin locations:**
- User plugins: `~/.claude/plugins/`
- Project plugins: Via settings.json configuration
- Marketplace plugins: Auto-managed by Claude Code

### MCP Server Setup

**Install MCP servers via `.mcp.json`:**

```json
{
  "mcpServers": {
    "github": {
      "command": "node",
      "args": ["/path/to/github-mcp-server.js"],
      "env": {
        "GITHUB_TOKEN": "${GITHUB_TOKEN}"
      }
    },
    "postgres": {
      "command": "docker",
      "args": ["run", "postgres-mcp-server"],
      "env": {
        "DATABASE_URL": "${DATABASE_URL}"
      }
    }
  }
}
```

**Manage MCP servers:**

```bash
# In the REPL
/mcp list
/mcp connect <server-name>
/mcp disconnect <server-name>
```

### LSP Server Setup

**Configure in `.lsp.json`:**

```json
{
  "typescript": {
    "command": "typescript-language-server",
    "args": ["--stdio"],
    "extensionToLanguage": {
      "ts": "typescript",
      "tsx": "typescriptreact"
    }
  },
  "python": {
    "command": "pyright-langserver",
    "args": ["--stdio"],
    "extensionToLanguage": {
      "py": "python"
    }
  }
}
```

Or install official LSP plugins:

```bash
/plugin install pyright-lsp@marketplace
/plugin install typescript-lsp@marketplace
/plugin install rust-lsp@marketplace
```

### Troubleshooting

**Command not found:**
- Check if component exists: `/help` (for commands)
- Verify file location: `.claude/commands/`, `.claude/agents/`, `.claude/skills/`
- Check file naming: Commands use filename, agents use `name` frontmatter field
- Restart session: Exit and relaunch Claude Code

**Plugin not loading:**
- Verify in settings.json: `"enabledPlugins": { "plugin-name": true }`
- Check plugin compatibility with Claude Code version
- Review plugin structure: Ensure `plugin.json` exists in `.claude-plugin/` directory
- Enable debug mode: `claude --debug plugin`

**Skill not being invoked:**
- Check description clarity: Claude matches semantically
- Verify skill location: `.claude/skills/<skill-name>/SKILL.md`
- Review frontmatter: Ensure `name` and `description` fields are present
- Check allowed-tools: Skill may be restricted from needed tools
